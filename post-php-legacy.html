PHP и jQuery: как поддерживать старый код и не бояться легаси

// фрагмент кода сюда

За 13 лет через мои руки прошло немало проектов, которым на момент, когда я к ним подключался, было уже по 5–8 лет. Полностью переписать такой код «правильно» почти никогда не разрешают — бизнес не готов останавливать продажи ради рефакторинга. Вот как я работаю с легаси на PHP без паники.

Сначала — не трогать то, что работает

Первое правило: если функция работает и покрыта реальными сценариями использования, не переписывать её «из принципа». Легаси-код обычно уже прошёл проверку боем — там учтены странные крайние случаи, о которых новый разработчик даже не подозревает.

Изоляция нового кода от старого

// новый функционал — в отдельном неймспейсе,
// не смешиваем со старыми глобальными функциями
namespace App\Modules\Promo;

class PromoService {
  public function apply($order) {
    // новая логика здесь, старый код не трогаем
  }
}

Новый функционал пишу в отдельных классах и неймспейсах, а со старым кодом соединяю через тонкий слой-адаптер. Так легаси постепенно окружается новым, более тестируемым кодом, а не переписывается целиком за один рывок.

jQuery — это не всегда плохо

Есть модный тезис «jQuery устарел, надо всё на современном стеке». На практике, если страница на 90% статична и там нужно повесить пару обработчиков и AJAX-запрос, jQuery решает задачу за десять строк без сборки, бандлеров и лишней инфраструктуры. Меняю стек только там, где это реально того стоит — например, в интерактивных формах с большим количеством состояний.

Три правила для спокойной работы с легаси

  • Перед изменением — понять, зачем код написан именно так, а не переписывать сходу
  • Любое изменение старого кода — с ручным тестированием сценариев, которые точно затронуты
  • Документировать не «как написано», а «почему написано так» — это экономит время следующему разработчику, часто это снова я сам через полгода

Легаси-код не враг — это накопленный опыт всех, кто работал над проектом до тебя. Задача не бороться с ним, а аккуратно встраивать новое рядом.
