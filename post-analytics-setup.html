Яндекс.Метрика и Google Analytics: минимальный набор для старта

// панель отчётов сюда

На большинстве проектов, куда я приходил, счётчики аналитики уже стояли — но настроены были только «для галочки»: считался общий трафик, и всё. Ниже — минимальный набор целей, который настраиваю в первую очередь на новом проекте.

Цель на отправку формы, а не на посещение страницы «спасибо»

Частая ошибка — считать конверсией просто визит на страницу благодарности. Если пользователь открыл эту страницу напрямую или вернулся на неё кнопкой «назад», конверсия засчитывается ложно. Настраиваю цель на само событие отправки формы через JavaScript-событие, а не на URL.

// отправка цели в Яндекс.Метрику по факту сабмита формы
form.addEventListener('submit', function() {
  ym(XXXXXXX, 'reachGoal', 'form_submit');
});

Вебвизор и карты кликов — не для красоты

Вебвизор Яндекс.Метрики за первую неделю обычно показывает больше, чем недельный отчёт по цифрам: где пользователи застревают, какие элементы принимают за кнопки, а какие важные ссылки никто не замечает. Смотрю записи сессий в первую неделю на каждом новом проекте — это быстро окупается.

Электронная торговля — если есть корзина

Для интернет-магазинов сразу настраиваю модуль электронной торговли и в Метрике, и в Google Analytics — тогда видно не просто «сколько было визитов», а на каком шаге воронки покупатели уходят: карточка товара → корзина → оформление → оплата.

Минимальный набор целей для старта

  • Отправка основной формы (заявка, обратная связь)
  • Клик по номеру телефона и по мессенджеру
  • Для магазина — добавление в корзину и оформление заказа
  • Глубина прокрутки на длинных посадочных страницах

Этого набора достаточно, чтобы через месяц уже можно было принимать решения по сайту на основе данных, а не ощущений — и только после этого имеет смысл настраивать более сложные сценарии.
