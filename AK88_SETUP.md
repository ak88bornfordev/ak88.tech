# Как подключить синхронизацию с телеграм-каналом AK88

Инструкция по настройке `sync_telegram.py`, чтобы посты из канала
`@ak88chanel` автоматически появлялись в разделе `ak88.html` на сайте.

---

## Шаг 1. Создайте файл `.env` с токеном

В папке сайта уже лежит файл-пример `.env.example`. Сделайте с него копию:

```bash
cp .env.example .env
```

Откройте `.env` любым текстовым редактором и впишите туда токен бота:

```
TELEGRAM_BOT_TOKEN=ваш_токен_бота
```

**Важно:**
- Файл `.env` — только у вас на компьютере. Он уже добавлен в `.gitignore`,
  поэтому даже если вы используете git, он не попадёт в репозиторий.
- Никогда не вписывайте токен в `sync_telegram.py`, `ak88.html` или любой
  другой файл сайта — только в `.env`.
- Если токен когда-либо случайно попадёт в публичное место (чат, репозиторий,
  скриншот) — сразу перевыпустите его через @BotFather → `/token`.

---

## Шаг 2. Проверьте, что бот — администратор канала

Бот `@ak88one_bot` должен быть добавлен в канал `@ak88chanel` именно как
**администратор** (не просто участник) — иначе Telegram не будет присылать
боту посты канала. Проверить: Настройки канала → Администраторы → убедиться,
что бот там есть.

---

## Шаг 3. Установите Python (если ещё не установлен)

Скрипту нужен Python 3.8 или новее, без дополнительных библиотек.

Проверить, установлен ли Python:

```bash
python3 --version
```

Если команда не найдена — установите Python с [python.org](https://www.python.org/downloads/)
(на Windows при установке обязательно поставьте галочку "Add Python to PATH").

---

## Шаг 4. Запустите скрипт вручную — первая проверка

Находясь в папке сайта (там же, где `sync_telegram.py` и `.env`):

```bash
python3 sync_telegram.py
```

Что должно произойти:
- Если в канале уже есть посты, опубликованные **после того**, как бот стал
  администратором — они появятся в `ak88.html`.
- Если новых постов нет, скрипт выведет: `Новых постов в канале нет.`
- Если появится ошибка — скопируйте текст ошибки (без токена) и пришлите мне,
  разберёмся.

Откройте `ak88.html` в браузере и убедитесь, что посты отображаются.

**Важный нюанс Telegram Bot API:** бот видит только те посты канала, которые
были опубликованы **после** его добавления в администраторы — более старые
посты подтянуть через этот способ нельзя.

## Шаг 5. Автоматизация — чтобы не запускать скрипт руками

Вручную запускать скрипт после каждого поста в канале неудобно. Есть два пути:

### Вариант А — если сайт лежит в репозитории на GitHub

1. Зайдите в репозиторий → **Settings → Secrets and variables → Actions →
   New repository secret**
2. Имя секрета: `TELEGRAM_BOT_TOKEN`, значение — сам токен
3. Создайте файл `.github/workflows/sync-telegram.yml` со следующим содержимым:

```yaml
name: Sync Telegram AK88
on:
  schedule:
    - cron: "*/15 * * * *"   # проверять каждые 15 минут
  workflow_dispatch: {}       # + кнопка запуска вручную на GitHub

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - name: Run sync script
        env:
          TELEGRAM_BOT_TOKEN: ${{ secrets.TELEGRAM_BOT_TOKEN }}
        run: python3 sync_telegram.py
      - name: Commit changes if any
        run: |
          git config user.name "telegram-sync-bot"
          git config user.email "actions@users.noreply.github.com"
          git add ak88.html telegram_state.json
          git diff --cached --quiet || git commit -m "Sync AK88 posts from Telegram"
          git push
```

4. Закоммитьте и запушьте этот файл — GitHub Actions сам возьмёт токен из
   секрета (в файле workflow токена нет, он появляется только во время
   выполнения на серверах GitHub).

Если сайт на GitHub Pages / Netlify / Vercel с автодеплоем при пуше —
изменения в `ak88.html` подхватятся автоматически после каждого коммита.

### Вариант Б — если сайт лежит на своём сервере / VPS

Настройте `cron`, чтобы запускать скрипт каждые 15 минут:

```bash
crontab -e
```

Добавьте строку (укажите свой путь к сайту):

```
*/15 * * * * cd /путь/к/сайту && /usr/bin/python3 sync_telegram.py >> sync.log 2>&1
```

`.env` при этом должен лежать рядом со скриптом прямо на сервере.

## Кратко: что и куда

| Что | Куда положить | Попадает в публичный доступ? |
|---|---|---|
| Токен бота | файл `.env` (локально) | Нет — в `.gitignore` |
| Токен бота (для автоматизации) | GitHub Secret `TELEGRAM_BOT_TOKEN` | Нет — виден только во время выполнения workflow |
| `sync_telegram.py`, `ak88.html` | обычные файлы сайта | Да, но токена там нет |
| `telegram_state.json` | локально / в репозитории (не критично, но лучше в `.gitignore`) | Не содержит токен, только ID последнего поста |
